Getting Started

Alkami Widget

Setup your widget

The widget template creates an MVC structured class library project that includes references to the required NuGet packages

First Steps

Some things will still need to be completed:

  1. Update the Alkami.Client and Alkami.Common packages. Look at "Installed Packages" under the NuGet Package Manager and update these versions to match the version of ORB you are targeting.
  2. Automated the execution of "install_widget.sql" script in the "Tools" directory, this will be inserting a few rows into a few tables of the DeveloperDynamic database related to widget. Check if your widget is present in the respective tables or not, if not, execute the script manually.
  3. Right click solution and select manage nuget packages
  4. Set the package source drop down to all
  5. Restore nuget packages, when prompted for credential put in your Alkami feed credentials
  6. Install nuget package Alkami.MicroServices.Accounts.Service.Client
  7. Install nuget package Alkami.MicroServices.Security.Client
  8. Rebuild project
  9. Once the project is built you should be able to navigate to your widget in the browser https://developer.dev.alkamitech.com/DonationWidget

Submitting your widget

Pre-compiling will only occur when you build your widget in "Release" mode. Before submitting your widget to Alkami, you will need to build your dll in "Release" mode

Updating Packages

Update the Alkami.Client and Alkami.Common NuGet packages as your staging environment is updated. Most times the libraries will be available before they are released to stage. Best practice is to stay in sync with your higher environments until a new Client or Common release is available for testing.
